# Contributing to slide-spec Docs

This page is about the docs site itself.

## Edit flow

1. Update the docs page or screenshot.
2. Run the docs build locally.
3. Review the output in the browser.
4. Keep the content aligned with the live schema and templates.

## Screenshot guidance

- Use declarative screenshot names.
- Keep docs screenshots in `docs/public/screenshots`.
- Prefer fixture-driven captures over hand-edited images.

## What to avoid

- speculative feature writeups
- stale screenshots
- long pages that make navigation harder
